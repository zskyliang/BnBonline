# BnB 游戏规则文档（用于 AI 躲泡训练）

本文档记录当前代码中已生效的「水泡爆炸命中 / 半身安全」规则，作为后续 AI 训练与调参基线。

## 1. 术语定义

- 安全区：当前时刻未被任意活动水柱覆盖的格子。
- 非安全区：当前时刻被活动水柱覆盖的格子。
- 活动水柱窗口：从水泡炸开开始，到水柱彻底消失结束的时间段。
- 半身安全：角色双脚中仅一只脚在非安全区，另一只脚在安全区时，整体判定为安全。

## 2. 爆炸窗口与非安全区规则

### 2.1 爆炸窗口时长（当前配置）

- 计算公式：`(ExplosionAnimEndFrame - ExplosionAnimStartFrame + 2) * ExplosionAnimFrameIntervalMs`
- 当前常量：
  - `ExplosionAnimStartFrame = 6`
  - `ExplosionAnimEndFrame = 13`
  - `ExplosionAnimFrameIntervalMs = 50ms`
  - `ExplosionForceDisposeMs = 900ms`（上限保护）
- 计算结果：`(13 - 6 + 2) * 50 = 450ms`
- 实际生效窗口：`min(450ms, 900ms) = 450ms`
- 可在运行时读取：`window.BNBExplosionUnsafeWindowMs`

### 2.2 区域判定

- 在爆炸窗口内，水柱覆盖到的格子全部属于非安全区。
- 未被覆盖的格子属于安全区。
- 活动窗口结束后，该水柱贡献的非安全区立即失效。

## 3. 每帧判定与命中规则

### 3.1 每帧检测范围

- 每 1 帧执行一次安全判定（玩家与 AI 都参与）。
- 仅当满足下列任一条件才检测：
  - 地图上存在泡泡，或
  - 仍存在活动爆炸窗口。
- 若两者都不满足：本帧不做爆炸判定，并重置连续命中计数。

### 3.2 连续命中规则

- 角色在某一帧被判定为非安全时：`ExplosionUnsafeFrameCount += 1`
- 角色在某一帧被判定为安全时：`ExplosionUnsafeFrameCount = 0`
- 当 `ExplosionUnsafeFrameCount >= 2` 时，判定为被水柱炸到，角色进入困泡（Trap）状态。

## 4. 半身安全判定细则

### 4.1 双脚采样方式

- 每帧对角色左右脚做格子采样：
  - 横向偏移：`±12`
  - 纵向偏移：`+8`（更宽容，减少贴下边缘时被判到下方危险格的概率）
- 采样函数：`Role.prototype.GetFootMapIDPair()`

### 4.2 判定真值表

- 左脚安全 + 右脚安全 => 安全
- 左脚非安全 + 右脚安全 => 安全（半身安全）
- 左脚安全 + 右脚非安全 => 安全（半身安全）
- 左脚非安全 + 右脚非安全 => 非安全

说明：只有“双脚同时在非安全区”才会累计命中计数。

## 5. 判定顺序（严格执行）

每帧按以下顺序执行：

1. 根据所有活动爆炸窗口构建当前非安全区范围。
2. 对每个角色计算左右脚落点并判定其是否安全（包含半身规则）。
3. 更新连续命中计数（安全重置为 0，非安全 +1）。
4. 若连续命中计数达到 2，则触发被炸与困泡。

## 6. 调试日志规则

- 游戏内提供「安全区判定日志」面板，并同步输出到控制台。
- 日志会输出：
  - 安全原因（双脚安全、半身安全、无敌、安全但未形成水柱等）
  - 非安全原因（双脚都在非安全区 + 连续命中计数）
  - 被炸原因（连续 >=2 帧非安全 + 攻击者）
- 日志为倒序显示（最新在上），用于训练过程实时观测。

## 7. AI 训练使用建议（基于当前规则）

- AI 必须按“每帧、双脚”维度评估自身状态，不能仅按角色中心点判断。
- 决策时优先避免“双脚同时进入非安全区”持续 2 帧。
- 在边界区域可利用半身安全过渡，但应避免连续帧双脚落入非安全区。
- 训练复盘需重点统计：
  - 连续命中触发前的 1~2 帧站位与目标格
  - 双脚格号变化（特别是横向移动）
  - 当帧非安全区覆盖与窗口剩余时间
